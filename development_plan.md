# План разработки Streamlit-приложения для BS_qwen.py

## Обзор проекта

Разработка интерактивного веб-интерфейса на базе Streamlit для скрипта анализа стабильности углеводородов `BS_qwen.py`. Приложение позволит загружать данные, настраивать все параметры анализа, запускать расчеты, визуализировать результаты и сохранять конфигурации.

---

## Этап 1: Подготовка окружения и анализ кода

### 1.1. Анализ существующего кода `BS_qwen.py`
- **Цель:** Полное понимание структуры, входных/выходных данных и зависимостей.
- **Действия:**
  - Изучить функцию `main()` или точку входа в скрипт.
  - Выделить все глобальные переменные и параметры, которые нужно вынести в GUI.
  - Определить функции, отвечающие за:
    - Загрузку данных (`pandas.read_excel`).
    - Предобработку (фильтрация выбросов, нормализация).
    - Расчеты (PCA, оптимизация, статистика).
    - Визуализацию (matplotlib/seaborn).
    - Сохранение результатов.
  - Выявить жестко заданные значения (hardcoded), которые станут параметрами интерфейса.
- **Результат:** Список всех параметров с типами данных, диапазонами допустимых значений и описаниями.

### 1.2. Подготовка зависимостей
- **Цель:** Обеспечить работу Streamlit и необходимых библиотек.
- **Действия:**
  - Создать файл `requirements.txt` с перечнем библиотек:
    - `streamlit`
    - `pandas`, `openpyxl` (для Excel)
    - `numpy`, `scipy`, `scikit-learn` (для расчетов)
    - `plotly` (для интерактивных графиков)
    - `matplotlib` (как бэкенд при необходимости)
  - Проверить совместимость версий библиотек с текущим кодом `BS_qwen.py`.
- **Результат:** Готовый файл `requirements.txt`.

### 1.3. Рефакторинг `BS_qwen.py` (Модуляризация)
- **Цель:** Сделать код пригодным для импорта в Streamlit-приложение.
- **Проблема:** Скрипты часто написаны как "плоский" код с глобальными переменными и вызовами `plt.show()`.
- **Действия:**
  - Обернуть основной поток выполнения в функцию `run_analysis(input_data, params_dict)`, которая принимает данные (DataFrame) и словарь параметров.
  - Убрать прямые вызовы `plt.show()` и `input()`. Вместо этого функции визуализации должны возвращать объекты фигур (Plotly Figure или matplotlib Figure).
  - Гарантировать, что функции не имеют побочных эффектов (не пишут файлы без явного вызова).
  - Добавить обработку ошибок (try/except) с возвратом понятных сообщений.
- **Результат:** Модуль `BS_qwen.py`, который можно импортировать как `import BS_qwen` и вызывать `BS_qwen.run_analysis(...)`.

## Этап 3: Реализация загрузки и предпросмотра данных

### 3.1. Виджет загрузки файла
- **Цель:** Позволить пользователю выбрать файл.
- **Действия:**
  - Добавить `st.file_uploader` с поддержкой `.xlsx`, `.xls`, `.csv`.
  - Реализовать кэширование функции загрузки через `@st.cache_data` для производительности.
  - Обработать случаи отсутствия файла или ошибки формата.

### 3.2. Валидация и предпросмотр
- **Цель:** Убедиться, что данные корректны перед запуском.
- **Действия:**
  - После загрузки прочитать файл в `pandas.DataFrame`.
  - Проверить наличие обязательных колонок (согласно логике `BS_qwen.py`).
  - Вывести предупреждение, если колонки отсутствуют.
  - Отобразить первые 5-10 строк данных через `st.dataframe` или `st.data_editor` (только для чтения).
  - Показать базовую статистику (количество строк, типы данных).
- **Результат:** Пользователь видит загруженные данные и уверен в их корректности.

---

## Этап 4: Разработка панели управления параметрами

### 4.1. Структурирование параметров
- **Цель:** Удобно разместить все настройки.
- **Действия:**
  - Сгруппировать параметры по смыслу (используя `st.expander`):
    - **Основные настройки:** Выбор колонок (HC, годы), пороги отсечения.
    - **Параметры методов:** Чекбоксы для включения/выключения конкретных методов анализа.
    - **Оптимизация:** Выбор алгоритма, количество итераций, размер популяции.
    - **Фильтрация выбросов:** Методы (Z-score, IQR), пороговые значения.
    - **Визуализация:** Настройки цветов, размеров точек, подписей.

### 4.2. Реализация виджетов ввода
- **Цель:** Создать элементы управления для каждого параметра.
- **Действия:**
  - Использовать соответствующие типы виджетов:
    - `st.number_input` для числовых значений (с мин/макс и шагом).
    - `st.selectbox` / `st.multiselect` для выбора из списка.
    - `st.checkbox` для булевых флагов.
    - `st.slider` для интуитивного выбора диапазонов.
    - `st.text_input` для имен файлов или пользовательских меток.
  - Привязать значения виджетов к переменным состояния (session_state), чтобы они не сбрасывались при перерисовке.
- **Результат:** Полностью настроенная панель параметров, сохраняющая состояние.

## Этап 5: Система сохранения и загрузки конфигураций

### 5.1. Формат хранения
- **Цель:** Позволить пользователю сохранять свои настройки.
- **Действия:**
  - Выбрать формат JSON для хранения конфигов (человекочитаемый и легкий).
  - Создать папку `saved_configs` в корне проекта.

### 5.2. Функционал сохранения
- **Цель:** Экспорт текущих настроек.
- **Действия:**
  - Кнопка "Сохранить конфигурацию" в сайдбаре.
  - Поле ввода имени конфига.
  - Функция, собирающая все значения из `st.session_state` в словарь и записывающая их в JSON-файл.
  - Уведомление об успешном сохранении (`st.success`).

### 5.3. Функционал загрузки
- **Цель:** Импорт ранее сохраненных настроек.
- **Действия:**
  - Выпадающий список с доступными конфигами (сканирование папки `saved_configs`).
  - Кнопка "Загрузить".
  - Функция, читающая JSON и обновляющая `st.session_state` соответствующими значениями.
  - Автоматическое обновление виджетов на экране.

### 5.4. Сброс настроек
- **Цель:** Быстрый возврат к значениям по умолчанию.
- **Действия:**
  - Кнопка "Сбросить все".
  - Очистка `st.session_state` и перезагрузка страницы.

## Этап 6: Интеграция логики анализа и запуск

### 6.1. Кнопка запуска
- **Цель:** Инициировать процесс расчета.
- **Действия:**
  - Большая кнопка "Запустить анализ" в основном блоке.
  - Блокировка кнопки, если файл не загружен.

### 6.2. Индикация прогресса
- **Цель:** Показать пользователю, что работа идет.
- **Действия:**
  - Использовать `st.spinner("Выполняется анализ...")` вокруг вызова функции.
  - Если в коде `BS_qwen.py` есть циклы, добавить передачу колбэка для обновления `st.progress` (опционально, если код позволяет).

### 6.3. Вызов функций анализа
- **Цель:** Передать данные и параметры в `BS_qwen.py`.
- **Действия:**
  - Собрать все параметры из UI в единый словарь `params`.
  - Вызвать основную функцию `results = BS_qwen.run_analysis(df, params)`.
  - Обработать исключения: если ошибка, вывести текст ошибки через `st.error` с трассировкой (для отладки).
  - Сохранить результаты в `st.session_state.results`, чтобы они не пропали при изменении других виджетов.

## Этап 7: Визуализация результатов

### 7.1. Переход на Plotly
- **Цель:** Сделать графики интерактивными (зум, ховер, сохранение).
- **Действия:**
  - Адаптировать функции построения графиков из `BS_qwen.py` (или создать новые обертки).
  - Заменить `matplotlib` на `plotly.express` или `plotly.graph_objects` там, где это возможно без потери смысла.
  - Для сложных графиков, которые трудно переписать, использовать `st.plotly_chart(fig_mpl)` (конвертация matplotlib в plotly через mpld3 или просто рендеринг статики, если интерактив не критичен). *Рекомендуется нативный Plotly.*

### 7.2. Организация вкладок результатов
- **Цель:** Логически разделить выводы.
- **Действия:**
  - Создать `st.tabs`:
    - **"Обзор данных":** Распределения, корреляционные матрицы (heatmap).
    - **"Многомерный анализ":** 3D PCA, t-SNE проекции.
    - **"Сравнение годов":** Графики изменений концентраций, дельты.
    - **"Таблица результатов":** Итоговая таблица с расчетными коэффициентами, стабильностью и т.д.
    - **"Отчет":** Текстовое резюме анализа.

### 7.3. Интерактивность таблиц
- **Действия:**
  - Отображение итоговых данных через `st.dataframe` с включенной сортировкой и фильтрацией.
  - Подсветка аномальных значений (conditional formatting).
